Topic 2: Execution Context & Call Stack
Understanding how JavaScript runs your code internally is critical to mastering advanced concepts like hoisting, closures, and async behavior.

üìå What is an Execution Context?
An execution context is the environment in which JavaScript code is evaluated and executed.

There are 3 types:

Global Execution Context (GEC)

Created when your script starts

One per program

this refers to the global object (like window in browsers)

Function Execution Context (FEC)

Created whenever a function is called

Each has its own arguments, local variables, and this

Eval Execution Context (rarely used, and discouraged)

üìå Two Phases of Execution
Every execution context goes through:

Creation Phase

Memory is allocated for variables and functions

var is hoisted (set to undefined)

let and const are hoisted but not initialized (temporal dead zone)

this is set

Execution Phase

Code is executed line by line

Variables are assigned values

Functions are invoked
